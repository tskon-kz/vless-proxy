# Файловый вотчер (`core/watcher.py`)

[English](../en/09-watcher.md)

## Назначение

Позволяет обновлять список прокси без использования Telegram-бота или API — достаточно положить файл `vless.txt` в директорию проекта.

## Поведение

**Логика работы с файлом:**

1. Файл не существует → ничего не делать, ждать
2. Файл пустой (0 байт или только пробелы) → ничего не делать, ждать пока появится контент
3. Файл содержит только комментарии (`#...`) или не-VLESS строки → ничего не делать
4. Файл содержит хотя бы одну `vless://` ссылку → загрузить, **удалить файл**

Файл удаляется после успешной загрузки. Это ключевое решение: при следующем перезапуске сервиса файла уже нет и состояние БД не перетирается.

Если загрузка упала с ошибкой — файл **не удаляется**, можно исправить и попробовать снова.

## Формат файла

```
# Комментарии начинаются с #, игнорируются
vless://uuid1@host1:443?security=reality&...#Название1
vless://uuid2@host2:443?security=tls&...#Название2

# Временно отключённый сервер:
# vless://uuid3@host3:443?...#Название3
```

## Использование

```python
watcher = FileWatcher(manager)
await watcher.load_once()          # загрузить файл при старте (если есть)
asyncio.create_task(watcher.run_forever())  # фоновый опрос
```

### `load_once() → bool`

Вызывается один раз при запуске сервиса. Если файл есть и содержит ссылки — загружает и удаляет. Возвращает `True` если загрузка была выполнена.

### `run_forever()`

Бесконечный цикл: каждые `FILE_CHECK_INTERVAL` секунд проверяет файл через `_check_file()`.

### `_check_file()`

Внутренний метод опроса:
- Вычисляет SHA-256 хэш файла
- Если файл пустой — возвращается без обновления хэша (на следующем такте снова проверит)
- Если хэш совпадает с предыдущим — пропускает
- Если хэш новый — читает и загружает

Хэш обновляется **только после успешного чтения непустого файла**. Пустой файл не фиксируется в хэше — это позволяет сначала создать файл (`touch vless.txt`), а потом заполнить его; вотчер не пропустит момент появления контента.

## Настройки

| Переменная | По умолчанию | Описание |
|---|---|---|
| `VLESS_FILE` | `./vless.txt` | Путь к файлу |
| `FILE_CHECK_INTERVAL` | `30` | Интервал проверки, сек |

## Сценарий обновления прокси через файл

```bash
# 1. Создать файл (можно заранее — вотчер дождётся контента)
touch vless.txt

# 2. Заполнить ссылками
echo "vless://..." >> vless.txt
echo "vless://..." >> vless.txt

# 3. Либо через скрипт
echo "vless://..." | bash scripts/update-proxies.sh

# Через FILE_CHECK_INTERVAL секунд файл будет прочитан и удалён
```
