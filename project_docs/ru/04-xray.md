# Управление xray-core (`core/xray.py`)

[English](../en/04-xray.md)

## `XrayProcessPool`

Управляет набором постоянных подпроцессов xray — по одному на каждый активный прокси.

### Ключевые методы

| Метод | Описание |
|---|---|
| `start_proxy(proxy_id, config, port?)` | Записать конфиг xray, запустить процесс, зафиксировать в БД |
| `stop_proxy(proxy_id)` | Завершить процесс xray, удалить запись из БД |
| `stop_all()` | Остановить все процессы (вызывается при завершении работы) |
| `get_process(proxy_id)` | Вернуть `ProcessRow` для прокси или `None` |
| `get_all_ports()` | Вернуть `{proxy_id: local_port}` для всех запущенных процессов |

### Выбор порта

Если порт не задан в `start_proxy()`, пул вызывает `storage.get_available_port()` — находит наименьший свободный порт в диапазоне `PROXY_PORT_START..PROXY_PORT_END`. Возвращает `None`, если все порты заняты.

## `write_xray_config(config, port, config_path)`

Генерирует JSON-конфиг xray: VLESS outbound + SOCKS5 inbound. Поддерживает все типы транспорта (TCP, WebSocket, gRPC) и слои безопасности (none, TLS, Reality).

## Временные процессы для проверки

`check_proxy()` в `core/health.py` запускает отдельный временный процесс xray (на порту из диапазона `19900+`) для каждой проверки — этот диапазон никогда не пересекается с пулом SOCKS5. После проверки временный процесс завершается вне зависимости от результата.
